# 中文摘要初稿

渗透测试能力评估是衡量大语言模型在网络安全场景中实际应用价值的重要环节，对于推动大语言模型在安全分析、攻防辅助和知识推理等任务中的可靠落地具有现实意义。现有通用评测基准难以覆盖渗透测试场景中的 ATT&CK 知识组织方式、技术关联关系和多步推理需求，现有安全测评方案也普遍存在题型单一、评估粒度不足以及自动评分成本较高等问题。为解决上述问题，本文围绕评测数据集构建、自动评分方法设计和测评平台实现三个方面开展研究。

本文的主要工作包括：

1. 针对渗透测试领域缺乏标准化知识评测数据的问题，设计并构建了一套基于 MITRE ATT&CK 的大语言模型渗透测试能力评测数据集。该数据集围绕事实验证、技术识别、技术关联分析、场景理解与推理等能力设计了单选、多选、判断、排序、场景单选、多步推理和简答推理等多种题型，并通过自动生成、统一复审、正式冻结和测试集抽样等流程提升了数据的一致性与可用性。
2. 针对不同题型难以使用统一标准进行自动评估的问题，设计并实现了一套面向客观题与推理题的分层自动评分方法。该方法对客观题采用规则匹配与标准答案比对，对推理题采用技术编号提取、别名归一化、目标匹配与部分得分判断相结合的评分机制，从而在保证评估效率的同时提高了评分结果的稳定性和细粒度。
3. 构建了一个面向多模型实验的渗透测试能力测评平台，实现了数据集管理、模型配置、API 资源池调度、批量评测执行、结果统计分析和可视化展示等功能。该平台能够支持多模型在统一任务设置下进行自动测评，为后续实验分析、模型横向比较和论文结果复现提供了完整支撑。

关键词：大语言模型；渗透测试；能力评测；MITRE ATT&CK；自动评分

# Abstract

Penetration testing capability evaluation is an important approach for assessing the practical value of large language models in cybersecurity scenarios, and it is of clear significance for promoting their reliable application in security analysis, attack-defense assistance, and knowledge reasoning tasks. Existing general-purpose benchmarks are insufficient to cover the ATT&CK-oriented knowledge organization, technical correlation relationships, and multi-step reasoning requirements involved in penetration testing scenarios. Meanwhile, current security evaluation schemes often suffer from limited question diversity, insufficient evaluation granularity, and relatively high scoring costs. To address these problems, this thesis conducts research from three aspects: benchmark dataset construction, automated scoring method design, and evaluation platform implementation.

The main work of this thesis is summarized as follows:

1. A standardized benchmark dataset for evaluating the penetration testing capability of large language models is designed and constructed based on MITRE ATT&CK. The dataset covers multiple ability dimensions, including fact verification, technique identification, technique correlation analysis, scenario understanding, and reasoning, and organizes them into several question forms such as single-choice, multiple-choice, judgment, sequencing, scenario-based single-choice, multi-step reasoning, and short-answer reasoning. In addition, a workflow consisting of automatic generation, unified review, formal freezing, and test-set sampling is adopted to improve data consistency and usability.
2. To address the difficulty of applying a unified evaluation standard to heterogeneous question forms, a layered automated scoring method for objective questions and reasoning questions is designed and implemented. For objective questions, the method uses rule matching and standard answer comparison. For reasoning questions, it combines technique identifier extraction, alias normalization, target matching, and partial-credit judgment, thereby improving scoring stability and granularity while maintaining evaluation efficiency.
3. A penetration testing capability evaluation platform for multi-model experiments is implemented. The platform supports dataset management, model configuration, API resource pool scheduling, batch evaluation execution, result aggregation and analysis, and visualized result presentation. It provides a complete experimental infrastructure for subsequent analysis, cross-model comparison, and reproduction of the thesis results.

Keywords: large language models; penetration testing; capability evaluation; MITRE ATT&CK; automated scoring

# 第一章 绪论

## 1.1 研究背景与意义

随着大语言模型在通用问答、代码生成、知识推理和任务规划等方向上表现出持续增强的能力，其在网络安全领域中的应用潜力也日益受到关注。在渗透测试场景中，大语言模型不仅可以辅助安全研究人员完成技术检索、攻击路径分析和日志解释等任务，还可能进一步参与漏洞利用思路生成、战术技术映射以及安全事件研判等高认知负荷工作。与此同时，网络安全场景天然具有较强的知识结构性、任务链条性和结果敏感性，这使得对大语言模型安全能力的评估不能停留在通用 benchmark 的粗粒度比较上，而需要结合领域知识体系构建更贴近实际需求的测评方法。

现有关于大语言模型能力评测的研究已经在综合问答、数学推理、代码理解和专业考试等任务中形成了较为丰富的基准体系，但面向渗透测试知识与能力的评估仍然存在明显不足。一方面，现有通用评测数据往往缺少与 MITRE ATT&CK 等安全知识框架的显式对应关系，难以反映模型对战术、技术及其上下位结构的掌握情况；另一方面，部分已有安全评测工作更偏向靶场执行、攻防代理或漏洞利用效果验证，对于知识理解、技术区分、工具映射和场景推理等基础能力的细粒度刻画仍然不够充分。此外，不同题型之间的评分逻辑差异较大，尤其是多步推理和简答推理题，若缺乏统一且可复现的自动评估机制，往往会增加实验成本并影响结果稳定性。

围绕上述问题开展研究具有明显的理论价值与应用意义。从研究层面看，构建一套面向渗透测试知识评测的标准化数据集，有助于推动该领域形成更可比、更可复现的模型评估基线；从方法层面看，设计兼顾效率与细粒度的自动评分方案，有助于降低正式实验的实施成本，提高大规模模型比较的可操作性；从应用层面看，搭建集数据管理、评测执行和结果分析于一体的测评平台，能够为后续模型筛选、能力诊断和安全场景部署提供直接支持。因此，开展基于知识评测的大语言模型渗透测试能力评估研究，既具有较强的现实需求，也具有明确的方法论意义。

## 1.2 国内外研究现状

围绕本文研究主题，相关工作主要可从大语言模型能力评测研究、网络安全领域大语言模型评测研究以及基于 ATT&CK 的安全知识组织与应用研究三个方面进行梳理。

### 1.2.1 大语言模型能力评测研究

大语言模型能力评测研究主要关注如何通过标准化任务与数据集衡量模型在知识问答、逻辑推理、专业理解和综合决策等方面的表现。现有工作已经形成了覆盖通用问答、数学推理、代码生成、法律医学考试以及多学科综合测试等多种方向的 benchmark 体系。这类研究通过构建统一题目集合、标准答案和评价指标，为不同模型之间的横向比较提供了基础，也推动了大语言模型能力研究从单点案例展示走向系统化、可复现的实验验证。

从评测方法上看，现有研究通常采用客观题自动判分、开放题 LLM 辅助评估或人工复核相结合的方式。对于选择、判断等封闭式题型，评分标准相对清晰，适合大规模自动测评；而对于涉及解释、推理和生成的开放式问题，则更依赖复杂的评价策略。随着模型能力不断增强，单纯依靠总体正确率已经难以全面反映模型差异，越来越多研究开始强调按照任务类型、知识维度和错误模式对模型能力进行拆解分析。

总体而言，现有代表性大语言模型评测研究已经较好地解决了通用任务标准化、模型横向比较和多维指标分析等问题，具有覆盖范围广、实验可复现性较强等优点。然而，这些研究大多面向通用场景或传统专业考试任务，对于渗透测试这类高度依赖领域知识组织、技术关联理解和场景推理能力的任务支持仍然有限，难以直接回答模型是否真正具备网络攻击知识理解与迁移能力。

### 1.2.2 网络安全领域大语言模型评测研究

面向网络安全领域的大语言模型评测研究近年来逐渐增多，研究内容涵盖漏洞分析、恶意代码理解、安全问答、攻防推理以及自主代理执行等多个方向。部分工作通过构建安全选择题、CTF 题目或漏洞案例来评估模型的基础安全知识，部分工作则通过靶场环境、攻击代理或自动化任务链来评估模型在真实攻防过程中的执行能力。这些研究表明，大语言模型在安全知识检索、脚本辅助生成和部分安全问答任务中已经展现出一定能力，但在复杂推理、上下文约束理解和稳定执行方面仍存在明显不足。

现有网络安全测评工作大致可以分为两类。一类侧重知识型评测，通常以问答、选择或案例判断为主，便于快速获得定量结果；另一类侧重行动型评测，更强调模型在连续任务中的计划、执行与反馈处理能力，能够更贴近真实安全任务环境。前者的优势在于标准化程度较高、评测成本较低，后者的优势在于场景真实性更强、能够反映模型在复杂任务中的综合表现。两类路径各有价值，但也分别存在粒度不够细或实验成本较高的问题。

总体来看，现有网络安全领域大语言模型评测研究已经证明了安全场景评测的必要性，也为模型在安全方向的应用提供了初步方法基础。然而，这些工作在知识组织体系、题型多样性、能力维度划分和统一自动评分方面仍有不足，尤其缺少围绕 MITRE ATT&CK 框架构建、能够同时覆盖基础识别与场景推理能力的细粒度评测方案。

### 1.2.3 基于 ATT&CK 的安全知识组织与应用研究

MITRE ATT&CK 作为当前应用广泛的对抗战术与技术知识框架，为安全知识表达、攻击行为建模和检测映射分析提供了统一语义基础。已有研究和实践工作常将 ATT&CK 用于安全事件标注、攻击链分析、检测规则构建、威胁情报组织以及安全培训等场景。相较于零散的术语集合，ATT&CK 具有层次清晰、语义稳定和可扩展性强的特点，因此也适合作为大语言模型安全能力评测的数据组织基础。

在 ATT&CK 的实际应用中，研究者通常关注战术、技术、子技术及其关系网络的建模与利用。一方面，这种层级结构有助于将抽象攻击目标与具体实施技术关联起来；另一方面，ATT&CK 中不同技术之间还存在用途相近、步骤衔接和跨战术协同等复杂关系，这些特征为能力评测提供了天然的知识图谱式组织方式。若将评测题目显式映射到 ATT&CK 节点及其关系上，便能够进一步分析模型在知识识别、关系理解和场景推理等方面的具体表现。

总体而言，基于 ATT&CK 的安全知识组织研究已经较好地解决了安全知识标准化表达和跨任务复用的问题，具有结构清晰、可映射性强和便于扩展等优点。但现有工作更多聚焦于检测、防御和情报组织等应用方向，围绕大语言模型渗透测试能力评估的数据集构建与自动测评机制仍相对缺乏，这也构成了本文开展研究的重要切入点。

## 1.3 研究内容与创新点

针对当前大语言模型渗透测试能力评测中存在的数据缺乏标准化、题型覆盖不足以及自动评分机制不统一等问题，本文围绕基于知识评测的大语言模型渗透测试能力评估展开研究，构建了基于 MITRE ATT&CK 的多题型评测数据集，设计了面向客观题与推理题的自动评分方法，并实现了一个支持多模型统一测评与结果分析的实验平台。全文重点关注模型对渗透测试知识的识别、区分、关联和推理能力，并通过多模型对比实验验证所提出评测方案的有效性。

本文的贡献可概括为以下两个方面：

1. 构建了一套面向大语言模型渗透测试能力评估的标准化数据集。该数据集以 MITRE ATT&CK 知识框架为基础，围绕事实验证、技术用途、工具映射、战术分类、防御检测、攻击场景理解、技术关联分析、跨战术关联分析、多步推理和简答技术判断等能力维度组织评测内容，并以单选、多选、判断、排序、场景单选、多步推理和简答推理等题型作为测量载体，通过 generated、reviewed、final 和 test 四个阶段的数据流转机制提升了数据质量与正式实验可复现性。
2. 设计并实现了一套面向多题型的自动评测方法与实验平台。针对客观题和推理题的差异，本文分别采用规则匹配与技术编号归一化相结合的评分机制，并在此基础上实现了支持多模型并行评测、API 资源池调度、结果动态写入和前端分析展示的测评平台，为后续模型能力比较与错误案例分析提供了统一实验基础。

# 第二章 相关研究

本章介绍与本文工作密切相关的研究基础，主要包括大语言模型能力评测研究、网络安全领域大语言模型评测研究以及基于 ATT&CK 的安全知识组织与应用研究三部分。

## 2.1 大语言模型能力评测研究

大语言模型能力评测是理解模型真实性能边界、分析模型优劣结构以及支撑模型选择的重要基础。随着大语言模型在多类任务中持续取得进展，针对其知识掌握、推理能力、任务泛化能力和领域适应性的评测研究也不断发展，逐步形成了从通用 benchmark 到专业 benchmark、从单一正确率到多维能力拆解的演进路径。

现有研究普遍通过构建标准化题目集合、统一输入输出格式和量化评价指标，对模型在不同任务中的表现进行横向比较。对于选择、判断等封闭式题型，研究通常采用自动判分；对于开放式问答、长链推理和复杂生成任务，则逐渐引入更精细的人工评估或模型辅助评估方式。与此同时，研究者也越来越重视对模型错误来源的分析，而不仅仅关注最终得分。

总体来看，现有大语言模型能力评测研究已经较好地解决了模型横向可比性、实验可复现性和多任务覆盖等问题，具有标准化程度高、分析框架较成熟等优点。然而，通用评测数据难以准确反映安全领域尤其是渗透测试任务中的知识结构特征与任务需求，这为面向特定领域的能力评测研究提出了新的要求。

## 2.2 网络安全领域大语言模型评测研究

网络安全领域的大语言模型评测研究主要围绕安全知识问答、漏洞分析、恶意代码理解、攻防推理与自主代理执行等方向展开。相较于通用领域，安全任务通常具有更强的专业术语密度、更复杂的上下文依赖关系以及更高的结果敏感性，因此仅依靠通用 benchmark 难以准确反映模型在安全场景中的实际可用性。

现有研究中，一部分工作采用安全题库、案例问答或 CTF 题目来评估模型的基础知识掌握程度，这类方法便于量化和复现；另一部分工作则将模型放入靶场、代理框架或连续任务环境中，观察其在真实攻防链条中的计划、执行和反馈处理能力，这类方法更贴近真实任务但成本更高、变量更多。两类研究分别从知识能力和行动能力两个角度揭示了模型在安全场景中的表现差异。

总体而言，现有安全评测研究已经证明了针对领域场景设计专门 benchmark 的必要性，也提供了若干可借鉴的任务组织方式和评价思路。然而，现有工作在知识组织的标准化、题型覆盖的丰富性、能力维度分析的细粒度以及自动评分的一致性方面仍有明显提升空间。

## 2.3 基于 ATT&CK 的安全知识组织与应用研究

MITRE ATT&CK 框架以战术、技术和子技术为核心，构建了较为系统的攻击知识表达体系，已经广泛应用于安全检测、攻击分析、威胁情报组织和安全培训等方向。由于其知识结构清晰、语义边界相对稳定、扩展方式规范，ATT&CK 也为构建标准化安全评测数据提供了良好基础。

在实际研究与应用中，ATT&CK 不仅能够表示单个攻击技术，还能体现不同技术之间的上下位关系、功能相似关系以及跨战术关联关系。这些关系对渗透测试任务尤为重要，因为渗透测试不仅要求识别单个技术点，还要求理解技术用途、步骤衔接与场景中的组合方式。因此，将 ATT&CK 引入大语言模型能力评测，不仅能够提升数据组织的规范性，也有助于构建更细粒度的能力分析框架。

总体来看，基于 ATT&CK 的知识组织方法已经在安全领域展现出较强的标准化优势与应用价值，但将其系统用于大语言模型渗透测试能力评估的数据构建与实验验证仍较少见。本文正是在这一基础上，将 ATT&CK 作为数据组织与能力映射的核心框架。

## 2.4 本章小结

本章从大语言模型能力评测、网络安全领域评测以及基于 ATT&CK 的安全知识组织三个方面对相关研究进行了梳理。现有研究为本文提供了重要的方法参考与问题背景，但在渗透测试知识评测的数据标准化、题型丰富性、自动评分一致性和多维能力分析方面仍存在不足。基于此，下一章将进一步介绍本文提出的评测数据构建方法、自动评分机制与平台实现方案。

# 第三章 基于知识评测的大语言模型渗透测试能力评估方法与系统设计

本章围绕本文提出的评测方案展开说明，重点介绍评测数据的组织方式、多题型数据集构建方法、自动评分机制以及测评平台的系统实现。与以往仅关注单一问答正确率或靶场任务结果的评测方式不同，本文试图构建一套覆盖基础知识识别、技术关系理解、场景分析与推理能力的细粒度评测体系，并通过统一的数据结构与评分流程提升实验可复现性。

## 3.1 引言

大语言模型在渗透测试场景中的表现并不取决于单一能力，而是由知识掌握、术语辨析、技术映射、上下文理解和多步推理等多个方面共同决定。若仅采用单一题型或粗粒度分数进行评估，往往难以揭示模型在不同知识层次上的真实差异。因此，本文将 MITRE ATT&CK 作为知识组织基础，将评测任务拆分为多类题型与能力维度，并在此基础上构建标准化数据集与统一自动评分流程。

从整体方法上看，本文提出的评测框架可分为数据构建层、评测执行层和结果分析层三个部分。数据构建层负责从 ATT&CK 知识框架中抽取战术、技术和子技术信息，并按照预设题型生成正式评测样本；评测执行层负责调用不同大语言模型完成统一测试任务，并依据题型差异进行自动评分；结果分析层则对评测结果按总体得分、题型表现和能力维度进行统计，从而支持模型间的横向比较与错误模式分析。该框架既强调知识组织的规范性，也强调实验执行与结果分析的一体化。

## 3.2 评测数据集构建方法

### 3.2.1 基于 ATT&CK 的知识组织方式

本文将 MITRE ATT&CK 作为评测数据构建的知识来源，核心原因在于该框架能够以统一语义组织渗透测试相关知识，并显式表达战术、技术和子技术之间的层级关系。相较于零散收集的安全题库，ATT&CK 具有结构清晰、术语稳定、知识覆盖较系统等特点，适合作为面向大语言模型评测的数据组织基础。

在具体组织方式上，本文首先从 ATT&CK 数据源中解析出战术、技术和子技术信息，并保留其编号、名称、描述及上下位关系。对于每一个知识点，除原始技术描述外，还进一步关注其功能用途、常见攻击场景、相关工具、可观测防御方式以及与其他技术的关联关系。这样做的目的在于避免题目仅停留在术语记忆层面，而是使数据集能够覆盖从基础识别到关系理解再到场景推理的不同评测目标。

为保证后续评测与分析的一致性，本文对数据字段进行了统一规范。正式数据集中的每条样本均包含题目编号、题型标记、能力维度、目标技术编号、题目内容、标准答案、测试提示词等核心字段。其中，`question_form` 用于表示题目形式，`capability_dimension` 用于标记该题目主要考察的能力维度，`question_type` 保留为兼容字段。通过显式保留 ATT&CK 编号与能力标签，后续实验可以直接按题型、按技术或按能力维度进行统计分析。

### 3.2.2 多题型评测数据设计

考虑到渗透测试能力本身具有明显的层次性，本文没有采用单一题型构建评测集，而是设计了单选题、多选题、判断题、排序题、场景单选题、多步推理题和简答推理题七类题型。不同题型所承担的评测目标并不完全相同：单选、多选与判断题主要用于检验基础知识识别与概念区分能力；排序题用于评估过程理解与步骤组织能力；场景单选题用于评估攻击场景中的知识迁移与情境判断能力；多步推理题和简答推理题则用于刻画模型在复杂关系分析和开放式回答中的表现。

在题型与能力维度的映射关系上，本文将正式评测划分为事实验证、技术关联分析、跨战术关联分析、多步推理、简答技术判断、技术用途、工具映射、战术分类、防御检测、攻击场景理解和场景技术识别等多个维度。不同题型并非简单一一对应某一维度，而是形成多对多关系。例如，单选题既可以用于技术用途判断，也可以用于战术分类；场景单选题既可以考察攻击场景理解，也可以考察场景中的技术识别；多步推理题则更多关注跨技术、跨步骤的信息组织与推断能力。这样的设计使得本文既能比较不同题型难度，也能进一步分析模型在不同能力维度上的优势与短板。

为提高数据质量与正式实验的一致性，本文在题目生成后引入了统一的数据流转机制。项目当前采用 `generated`、`reviewed`、`final` 和 `test` 四级目录对数据集生命周期进行管理。其中，`generated` 目录保存初步生成样本，`reviewed` 目录保存经统一模型复审后的样本，`final` 目录保存冻结后的正式实验数据，`test` 目录则保存从正式数据中抽取出的少量测试样本，用于验证评分脚本与流程是否稳定。这种分层管理方式使得正式实验数据与调试验证数据相互隔离，有利于后续结果复现和数据集版本控制。

### 3.2.3 数据生成、复审与冻结机制

在数据生成阶段，本文基于 ATT&CK 知识点和预设题型模板，调用大语言模型自动生成候选题目。为避免题目在内容上过于单一，生成过程不仅要求模型给出题面和答案，还要求其结合目标技术的描述信息生成具有一定干扰性的选项、解释文本和标准化测试提示。对于推理类题型，则进一步要求模型在问题设置中引入场景上下文、技术关系或多步骤信息，以保证题目能够真实考察链式理解能力。

在复审阶段，本文采用统一的复审模型对候选题目进行再次检查。复审重点包括题型格式是否合规、标准答案是否明确、题面与目标技术是否一致、解释文本是否存在事实性偏差等。对于推理类题目，还需额外检查题目是否真正要求模型进行技术分析，而不是退化为表面关键词匹配。经过复审的题目被写入 `reviewed` 目录，并保留必要的复审结果记录，便于后续定位问题来源。

在冻结阶段，本文将已经通过复审的数据集进一步整理为正式实验版本，并写入 `final` 目录。冻结的意义在于固定实验输入，避免后续在正式模型比较过程中因数据集持续变化而导致结果不可比。与此同时，本文还从冻结后的正式数据中抽取了少量样本写入 `test` 目录，用于在正式实验前验证评分逻辑是否合理，尤其用于检查多步推理题和简答推理题的答案提取、技术编号归一化与部分得分机制是否运行正常。

## 3.3 自动评分方法与评测流程

### 3.3.1 客观题评分方法

客观题包括单选、多选、判断、排序和部分场景单选题。这类题目的共同特点是答案空间相对封闭，因而适合采用规则化自动评分。本文在模型测试时为客观题构造统一提示词，要求模型尽可能直接返回选项字母、排序结果或简短结论，以减少后处理歧义。评分阶段通过对模型输出进行格式化提取、去除冗余文本和标准答案对比，判断题目是否答对，并记录对应得分。

对于单选题和判断题，评分相对直接，只需比较提取出的单一答案是否与标准答案一致。对于多选题和排序题，本文在比较时要求答案集合或序列与标准答案严格匹配，从而保证客观题评分标准的一致性。对于场景单选题，虽然题干中带有上下文描述，但最终输出仍然为封闭式选项，因此仍归入客观题评分路径。该方法具有实现简单、评分速度快和大规模实验成本低等优点，适合处理正式数据集中的大多数基础题型。

### 3.3.2 推理题评分方法

相较于客观题，多步推理题和简答推理题的回答形式更加开放，若仅采用字符串完全匹配，往往会低估模型真实能力。因此，本文针对推理题设计了更细粒度的自动评分方法。该方法首先从模型回答中提取 ATT&CK 技术编号、技术名称或关键别名信息，然后对不同写法进行归一化处理，并与题目标注的目标技术集合进行匹配。在匹配过程中，不仅考虑是否出现目标技术，还考虑是否存在错误替代、遗漏或明显无关输出。

对于多步推理题，评分重点在于模型是否能够在多步骤场景中识别出关键技术链条，因此通常采用目标技术命中情况来决定得分。若模型能够完整识别题目要求的关键技术，则给予满分；若仅命中部分技术，则按照预设规则给予部分得分；若输出技术明显偏离目标场景，则记为低分或零分。对于简答推理题，本文同样关注目标技术是否被正确识别，但同时也会观察回答是否包含与题意相符的分析内容。由于正式实验中需要兼顾效率与一致性，本文最终仍以技术提取与匹配结果作为主要得分依据，而不采用完全依赖 AI judge 的自由裁量式打分方式。

这种设计的优点在于，一方面能够比严格字符串匹配更公平地反映模型在推理题上的表现，另一方面又保留了较高的实验可复现性。前期使用测试集对评分逻辑的检查表明，经过技术编号提取、别名归一化与部分得分判断后，MSR 和 SAR 题型的自动评估结果能够较稳定地反映模型回答质量，因此可以用于后续正式实验。

### 3.3.3 统一评测执行流程

在正式评测过程中，本文将不同模型放入统一执行流程中进行比较。首先，由前端或任务配置模块确定待测模型列表和参与实验的数据集文件；其次，后端根据配置依次调度模型调用任务，对每个模型执行完整数据集测试；随后，根据题型选择对应评分路径，实时将题目级得分写入本地结果文件；最后，对单模型结果进行汇总统计，输出总体准确率、平均得分、题型表现和能力维度表现等信息。

为了适应多模型实验场景，项目实现中采用了 API 资源池调度机制。API key 与模型列表不再简单一一绑定，而是作为可配置资源统一管理。当可用 API 数量多于或等于参与测评模型数量时，不同模型可并行启动完整评测任务；当 API 数量不足时，系统将优先保证单个模型完整跑完一轮任务后再释放资源给其他模型，从而降低资源竞争对实验稳定性的影响。与此同时，评测结果会动态写入本地，以避免长时间实验因中断而导致全部结果丢失。

## 3.4 评测平台设计与实现

### 3.4.1 平台总体架构

为支撑正式实验的批量执行与可视化分析，本文在评测方法基础上实现了一个面向多模型实验的 Web 测评平台。平台整体采用前后端分离较弱的一体化实现思路：前端负责数据集选择、模型配置、API 资源管理、任务状态展示与结果分析展示，后端负责数据管理、任务调度、模型调用、自动评分和结果写入。平台通过 HTTP 接口完成前后端之间的数据交换，并以本地文件形式保存实验结果和分析报告。

从系统结构上看，平台主要包含四类核心模块。第一类为数据管理模块，负责读取正式数据集、维护上传数据与标准目录结构；第二类为任务执行模块，负责根据实验配置调度模型调用与进度更新；第三类为自动评分模块，负责根据题型分发评分逻辑，并生成题目级与汇总级结果；第四类为结果分析模块，负责将评测结果按模型、题型和能力维度进行聚合，供前端进一步展示。由于平台服务于论文实验，因此系统设计更强调结果一致性、实验可追溯性和使用流程的清晰性，而不是通用产品化功能的复杂扩展。

### 3.4.2 前端交互设计

平台前端围绕“实验配置 - 任务运行 - 结果分析”这一主线进行组织。页面左侧为实验控制区，主要用于选择参与本次实验的数据集；右侧上方为模型配置区与运行面板，负责管理参与测评的模型和 API 资源池；右侧下方为结果分析区，用于展示测评完成后的总体结果、题型结果和维度结果。整个界面采用浅色学术风格设计，以简洁布局和结构化信息展示为主，尽量减少与实验本身无关的说明性文本。

在模型配置方面，平台当前支持默认模型列表与自定义模型补充，模型以“显示名 + model_id”的形式统一管理。API key 作为独立资源池维护，支持按行增删和统一连通性测试。运行面板中不再只展示简单的单任务进度，而是通过任务明细表展示模型与数据集之间的映射关系及完成进度，使用户能够直观看到每个模型当前处理的数据集状态。结果分析区则重点面向总体得分、题型对比和能力维度差异等信息展开展示，为论文实验结果整理提供直接支持。

### 3.4.3 后端实现与结果管理

后端实现中，平台围绕统一评测流程提供若干核心接口，包括数据集获取、模型配置获取、文件上传、任务启动、进度查询和结果获取等。实验启动后，后端会读取所选数据集与参与模型配置，构建具体任务，并根据 API 资源池状态分配可用调用资源。在模型调用过程中，系统支持超时重试与错误信息记录，以降低网络波动和第三方接口不稳定对正式实验的影响。

结果管理方面，平台按照标准目录结构将实验输出写入 `results/evaluations/`、`results/reviews/` 和 `results/analysis/` 等目录。正式模型评测结果被写入 `results/evaluations/`，其中既包含题目级回答记录，也包含汇总后的 `final_report` 信息，便于后续论文统计与分析。评审日志与质量检查结果被写入 `results/reviews/`，分析性报告则写入 `results/analysis/`。这种按任务类型分目录存储的方式有助于在多轮实验并行开展时保持文件命名与结果来源的清晰性。

## 3.5 本章小结

本章从评测数据集构建、自动评分方法与平台实现三个方面介绍了本文提出的整体评测方案。首先，本文以 MITRE ATT&CK 为知识组织基础，构建了覆盖七类题型和多个能力维度的标准化评测数据集，并通过生成、复审、冻结和测试集抽样机制提高了数据质量与实验可复现性；其次，针对客观题和推理题的差异，本文设计了分层自动评分方法，在保证效率的同时兼顾了对推理题回答质量的细粒度刻画；最后，本文实现了一个支持多模型实验执行、API 资源池调度和结果分析展示的评测平台，为后续实验比较与结果分析提供了统一实现基础。下一章将在此基础上进一步给出实验设置与结果分析。

# 第四章 实验与分析

本章基于第三章构建的正式评测数据集与统一测评平台，对不同大语言模型在渗透测试知识评测任务中的表现进行实验分析。考虑到本文的核心目标在于评估模型在渗透测试知识场景中的能力结构，而非仅对题型作表面区分，因此本章分析遵循“能力维度优先、题型类型辅助”的组织思路。实验重点关注三个层面：第一，不同模型在全体测试集上的总体表现；第二，不同模型在能力维度上的结构性差异；第三，不同模型在题型层面的表现特征与误差模式。在此基础上，进一步归纳模型在渗透测试知识理解任务中的主要优势与不足。

## 4.1 实验设置

### 4.1.1 实验数据集

正式实验使用本文构建并冻结后的 `final` 数据集。数据集共包含 7037 道题，首先围绕事实验证、技术关联分析、跨战术关联分析、多步推理、简答技术判断、技术用途、工具映射、战术分类、防御检测、攻击场景理解和场景技术识别等多个能力维度进行组织。这些维度分别对应基础事实识别、关系理解、场景定位和开放式推理等不同层次的评测目标，因此能够较全面地刻画模型在渗透测试知识任务中的能力结构。

在能力维度之下，本文进一步引入判断题、多选题、多步推理题、简答推理题、单选题、排序题和场景单选题七类题型作为具体实现载体。其中，单选题数量最多，共 4173 题，占全部样本的一半以上；判断题 832 题；多选题 422 题；排序题 412 题；场景单选题 405 题；多步推理题 392 题；简答推理题 401 题。这样的设计并非强调题型本身的重要性，而是借助不同题型在回答形式和任务约束上的差异，使各能力维度得到更充分的测量。

### 4.1.2 参评模型与实验流程

本次正式实验选取五个代表性模型参与测评，分别为 Gemini 3.1 Flash Lite、DeepSeek V3.2、Qwen 3.5 Flash、GPT-4o Mini 和 Doubao Seed 1.6 Flash。所有模型均通过统一调用接口完成测试，并使用同一批正式冻结数据集、同一套提示格式和同一套评分逻辑进行评估，以保证实验结果的可比性。

实验执行过程中，平台按照“模型完整完成全部题目”的原则进行任务调度。对于每个模型，系统依次读取全部正式数据文件并生成题目级记录；对于客观题，采用规则匹配与标准答案比对方式评分；对于多步推理题和简答推理题，则采用技术编号提取、别名归一化与目标技术匹配的方式计算得分。测评过程中所有题目级结果会动态写入本地结果文件，并在任务结束后汇总为模型级报告，用于后续统计分析。

### 4.1.3 评价指标

本文主要采用准确率和平均得分两个指标衡量模型表现。准确率用于表示模型完全答对题目的比例，适合反映模型在封闭式题型上的直接正确性；平均得分则用于反映模型在允许部分得分题型中的整体表现，尤其适用于多步推理题和简答推理题这类开放程度较高的任务。对于推理类题型，平均得分能够比单一准确率更细致地呈现模型是否部分识别出关键技术信息。

在统计层面，本文从三种粒度分析实验结果：其一是总体粒度，即统计模型在全部 7037 道题上的准确率与平均得分；其二是能力维度粒度，即分析模型在不同知识维度上的优势与短板；其三是题型粒度，即将题型表现作为辅助视角，用于解释模型在不同回答约束条件下的稳定性差异。通过这三层分析，可以更全面地理解模型在渗透测试知识评测任务中的表现特征。

## 4.2 实验结果与分析

### 4.2.1 总体性能对比

五个模型在正式数据集上的总体实验结果表明，不同模型之间已经表现出较为明显的能力结构差异。总体上，Gemini 3.1 Flash Lite 的综合表现最好，准确率达到 84.54%，平均得分达到 87.25%；DeepSeek V3.2 和 Qwen 3.5 Flash 的表现接近，准确率分别为 83.06% 和 83.43%，平均得分分别为 86.32% 和 86.14%；GPT-4o Mini 的总体表现略低于前三者，准确率为 80.87%，平均得分为 83.59%；Doubao Seed 1.6 Flash 的整体表现最低，准确率为 71.39%，平均得分为 73.83%。

如果仅从总体排序观察，可以得到各模型的相对位置；但结合后续能力维度结果来看，这一排序的关键并不在于某一单独题型的偶然优势，而在于模型是否能够同时兼顾基础知识映射、技术关系理解与场景化分析等多类能力。Gemini 3.1 Flash Lite 的优势主要体现为整体表现较为均衡，在多数基础能力维度上均保持稳定高分；DeepSeek V3.2 和 Qwen 3.5 Flash 虽然总分略低，但在个别能力维度上具备较强竞争力；GPT-4o Mini 则呈现出一定的结构性偏科特征，即在部分复杂推理任务中表现突出，但在其他维度上的失分拉低了总体成绩；Doubao Seed 1.6 Flash 在多类能力任务上均存在相对明显短板。

因此，本节的总体结果更适合作为能力分析的入口，而不能替代对模型内部能力分布的进一步讨论。只有结合能力维度与典型题型结果，才能更准确地解释不同模型在渗透测试知识场景中的真实表现。

### 4.2.2 不同能力维度表现分析

从能力维度角度看，各模型在事实验证、技术用途、工具映射、战术分类、防御检测和攻击场景理解等维度上的表现整体较好。以 Gemini 3.1 Flash Lite 为例，其在技术用途、工具映射、战术分类、防御检测和攻击场景理解等维度上的准确率均在 96% 以上；DeepSeek V3.2 在技术用途和防御检测上的准确率分别达到 98.92% 和 97.37%；Qwen 3.5 Flash 在防御检测和战术分类上的准确率也达到 97.60% 和 97.01%。这说明当任务主要涉及较明确的 ATT&CK 知识映射、技术用途判断或防御检测理解时，模型普遍能够给出较稳定回答。

然而，在技术关联分析、跨战术关联分析和简答技术判断等维度上，各模型表现明显下降。其中，技术关联分析是所有模型的共同薄弱项，Gemini 3.1 Flash Lite、DeepSeek V3.2、Qwen 3.5 Flash、GPT-4o Mini 和 Doubao Seed 1.6 Flash 的准确率分别仅为 28.26%、26.38%、30.29%、28.12% 和 26.38%。这意味着模型在面对相近技术之间的联系与区分时，容易出现判断不稳或关联不足的问题。

跨战术关联分析维度的结果略好于技术关联分析，但整体仍低于多数基础维度。五个模型在该维度上的平均得分介于 55.76% 至 71.18% 之间，其中 DeepSeek V3.2 表现最好，平均得分达到 71.18%，GPT-4o Mini 次之，为 68.82%。这一结果表明，部分模型已经具备一定跨战术推理能力，但整体上仍难以稳定处理涉及多层 ATT&CK 关系的复杂判断任务。

在简答技术判断维度上，模型差异尤为明显。Gemini 3.1 Flash Lite 的平均得分最高，为 43.39%；DeepSeek V3.2 次之，为 39.53%；Qwen 3.5 Flash、GPT-4o Mini 和 Doubao Seed 1.6 Flash 分别为 29.30%、18.33% 和 3.87%。这说明开放式技术判断不仅是整体难点，而且不同模型在这一维度上的能力差异比基础题型更大。换言之，总体得分接近的模型，在开放式技术分析任务上仍可能表现出显著区别。

此外，场景技术识别维度也呈现出一定差异。Qwen 3.5 Flash 在该维度上得分最高，为 88.40%，DeepSeek V3.2 为 86.17%，Gemini 3.1 Flash Lite 为 81.98%，GPT-4o Mini 为 80.74%。这一现象说明，在给定攻击上下文后，不同模型对关键技术点的定位能力并不完全一致。Qwen 3.5 Flash 在场景技术识别上的优势，与其在场景单选题上的较好表现是一致的。

总体而言，能力维度分析表明：第一，基础知识映射型任务已经不是当前模型的主要瓶颈；第二，技术关联分析和开放式技术判断是更能拉开模型差距的关键维度；第三，不同模型在场景理解与跨技术分析上的能力结构并不完全一致，因此仅依靠总体得分难以准确判断其在渗透测试知识任务中的真实优势方向。

### 4.2.3 不同题型表现分析

从题型角度看，各模型在封闭式题型和开放式题型上的表现差异十分明显。首先，在判断题和单选题上，多数模型都取得了较高准确率。Gemini 3.1 Flash Lite 在判断题和单选题上的准确率分别达到 97.24% 和 97.27%；DeepSeek V3.2 分别达到 93.27% 和 96.50%；Qwen 3.5 Flash 分别达到 93.39% 和 96.93%。这表明，对于基础事实识别、技术用途判断和较明确的术语辨析任务，当前主流模型已经具备较强能力。

其次，在场景单选题和多步推理题上，各模型仍保持了相对较高的得分，但差异开始显现。GPT-4o Mini 在多步推理题上的平均得分最高，达到 95.50%，表明其在需要跨步骤整合信息的任务中具备较强表现；Gemini 3.1 Flash Lite 和 Doubao Seed 1.6 Flash 的多步推理平均得分也分别达到 94.65% 和 92.78%；DeepSeek V3.2 和 Qwen 3.5 Flash 则分别为 92.52% 和 91.92%。在场景单选题上，Qwen 3.5 Flash 的准确率最高，达到 88.40%，说明其在带有明确上下文约束的封闭场景判断中表现较强。

再次，在多选题和排序题上，所有模型的表现都显著低于判断题和单选题。多选题平均得分大致分布在 43% 至 53% 之间，排序题平均得分大致分布在 43% 至 53% 之间。这表明，当题目需要模型同时识别多个正确项，或者要求模型理解攻击过程中的顺序关系时，错误率会明显上升。相比基础单项判断，多选与排序任务对模型提出了更高的细粒度区分要求。

最值得关注的是简答推理题。该题型的准确率整体偏低，Gemini 3.1 Flash Lite、DeepSeek V3.2、Qwen 3.5 Flash、GPT-4o Mini 和 Doubao Seed 1.6 Flash 的准确率分别仅为 37.16%、30.17%、19.45%、12.72% 和 2.49%；平均得分分别为 43.39%、39.53%、29.30%、18.33% 和 3.87%。这一结果表明，开放式技术判断仍然是当前模型在渗透测试知识评测中的主要难点。即使模型能够在部分回答中提及相关技术，其输出仍然容易出现识别不完整、概念混淆或关键技术遗漏等问题。

总体来看，题型分析表明模型在“基础识别型任务”上的表现已经较为成熟，但在“多目标辨析型任务”和“开放式技术判断型任务”上仍存在明显不足。尤其是简答推理题与多选题的结果说明，当前模型虽然能够在单点知识问答中取得较高正确率，但在需要更细粒度输出约束和更强技术关系理解的任务中，能力仍未完全稳定。

## 4.3 结果讨论

### 4.3.1 模型能力结构特征

综合总体结果、能力维度结果以及题型结果可以发现，当前模型在渗透测试知识评测中的能力结构呈现出较明显的“映射识别能力较强、复杂关系分析与开放式判断能力较弱”特征。无论是 Gemini 3.1 Flash Lite、DeepSeek V3.2 还是 Qwen 3.5 Flash，在事实验证、技术用途、工具映射、防御检测和战术分类等维度上都表现稳定，但在技术关联分析、跨战术关联分析以及简答技术判断等维度上失分明显。这说明当前模型更擅长处理边界清晰、映射关系明确的知识任务，而在需要综合辨析、跨节点联想和精确输出关键技术的任务中仍然容易出现偏差。

从模型个体差异来看，Gemini 3.1 Flash Lite 的优势在于综合表现均衡，在多数基础能力维度上都保持高分，因此总体成绩最优；DeepSeek V3.2 在跨战术关联分析和简答技术判断上的表现相对更好；Qwen 3.5 Flash 在场景技术识别与攻击场景理解相关任务上表现突出；GPT-4o Mini 虽然总体分数略低，但在多步推理任务上取得了全体模型中的最高平均得分，说明其在链式整合信息方面具有一定优势；Doubao Seed 1.6 Flash 则在多数关键能力维度上与前四个模型存在差距，尤其是在开放式技术判断任务上表现较弱。由此可见，不同模型在渗透测试知识任务中并非简单强弱关系，而是具有不同的能力分布特征。

### 4.3.2 典型误差模式分析

结合实验结果，可以将当前模型在渗透测试知识评测中的主要误差归纳为三类。第一类是基础概念混淆错误，即模型能够识别题目所属的大致攻击阶段，但无法稳定区分功能相近或语义接近的具体技术，这类错误在技术关联分析和多选题中表现尤为明显。第二类是关系链条不完整错误，即模型虽然能够识别部分关键技术，但在多步场景中遗漏必要步骤或跨战术关联节点，这类错误主要出现在多步推理和跨战术关联分析任务中。第三类是开放式回答约束不足错误，即模型在简答题中给出较宽泛描述，却没有准确落到题目要求的目标技术编号或技术名称上，这也是简答推理题准确率整体偏低的重要原因。

这些误差模式说明，当前模型的主要问题并不只是“完全不知道答案”，而更多体现为“知道相关知识但输出不够精确”或“能够识别局部信息但难以形成完整链条”。这也从侧面说明本文采用平均得分与技术提取相结合的推理题评分方式是必要的，因为若仅采用完全正确与完全错误的二元判断，将难以充分反映模型在复杂题型中的真实能力层次。

## 4.4 本章小结

本章基于正式冻结数据集，对五个大语言模型在渗透测试知识评测任务中的表现进行了系统分析。实验结果表明，Gemini 3.1 Flash Lite 在总体表现上最优，DeepSeek V3.2 与 Qwen 3.5 Flash 紧随其后，GPT-4o Mini 在多步推理任务中表现突出，而 Doubao Seed 1.6 Flash 的整体成绩相对较弱。更重要的是，能力维度分析进一步显示，当前模型在事实验证、技术用途、工具映射、防御检测等映射识别类任务上已经表现稳定，但在技术关联分析、跨战术关联分析、简答技术判断等更强调结构化理解与开放式表达的任务上仍存在明显不足。题型结果则从回答约束形式的角度对上述现象进行了补充说明。上述结果表明，面向渗透测试知识场景的细粒度评测不仅能够揭示模型总体能力差异，更能够识别不同模型在具体能力结构上的优势和短板。

# 第五章 总结与展望

## 5.1 全文总结

随着大语言模型在安全问答、漏洞分析、攻防辅助和自动化代理等场景中的应用不断增多，如何对其在渗透测试知识任务中的真实能力进行系统、细粒度且可复现的评估，已经成为模型研究与应用部署中的一个关键问题。针对现有通用评测难以充分覆盖网络安全知识结构、现有安全评测缺乏统一能力划分以及复杂推理题型自动评分不足等问题，本文围绕基于知识评测的大语言模型渗透测试能力评估展开了较为系统的研究。

本文首先构建了一套面向渗透测试知识评测的标准化数据集。该数据集以 MITRE ATT&CK 知识框架为基础，围绕事实验证、技术用途、工具映射、战术分类、防御检测、攻击场景理解、技术关联分析、跨战术关联分析、多步推理和简答技术判断等能力维度组织评测内容，并通过单选、多选、判断、排序、场景单选、多步推理和简答推理等题型作为测量载体，实现了对模型不同层次能力的覆盖。与此同时，本文设计了 generated、reviewed、final 与 test 四阶段数据流转机制，使数据集在生成、审查、冻结和调试使用之间形成了相对规范的闭环流程，为后续实验的稳定执行与结果复现提供了基础。

在评测方法方面，本文针对不同题型的输出特点设计了统一的自动评分流程。对于判断、单选、多选、排序和场景单选等客观题，采用规则匹配与标准答案比对的方式进行评分；对于多步推理和简答推理等开放式题型，则采用技术编号提取、别名归一化与目标技术匹配相结合的方式计算得分，从而在保证自动化执行效率的同时，尽可能保留对部分正确回答的区分能力。在此基础上，本文进一步实现了一个支持多模型统一调度、动态结果写入和前端可视化分析的评测平台，使正式实验、过程跟踪与结果展示能够在同一系统内完成。

在实验部分，本文选取 Gemini 3.1 Flash Lite、DeepSeek V3.2、Qwen 3.5 Flash、GPT-4o Mini 和 Doubao Seed 1.6 Flash 五个代表性模型进行了正式评测。实验结果表明，不同模型在渗透测试知识任务中的表现已经呈现出较为明显的能力结构差异。总体上，Gemini 3.1 Flash Lite 表现最优，DeepSeek V3.2 与 Qwen 3.5 Flash 紧随其后，GPT-4o Mini 在多步推理任务中体现出较强优势，而 Doubao Seed 1.6 Flash 的整体表现相对较弱。进一步的能力维度分析显示，当前模型在事实验证、技术用途、工具映射、防御检测等映射识别类任务上已经表现较为稳定，但在技术关联分析、跨战术关联分析和简答技术判断等更强调结构化理解与开放式表达的任务上仍存在明显短板。上述结果说明，面向渗透测试知识场景的细粒度评测不仅能够揭示模型总体水平差异，更能够更准确地刻画模型的优势能力与薄弱环节。

总体而言，本文完成了从数据集构建、评分方法设计到平台实现和实验验证的一体化工作，形成了一套面向大语言模型渗透测试知识能力评估的基础方案。该研究为后续开展更大规模、更高复杂度的安全能力测评提供了可复用的数据与系统基础，也为模型选型、能力诊断和安全场景应用评估提供了较为直观的实验依据。

## 5.2 不足与展望

尽管本文已经在数据集构建、自动评分和平台实现方面进行了较为完整的探索，但仍然存在一些不足，有待在后续研究中进一步完善。

首先，本文当前的数据集仍然以知识评测为核心，虽然已经通过能力维度划分和多题型设计增强了任务复杂性，但整体上仍属于相对受控的离线评测范式。对于真实渗透测试过程中涉及的长链规划、工具交互、环境反馈处理和多轮策略修正等能力，本文尚未进行更深入的动态任务建模。因此，后续研究可以在现有知识评测基础上，进一步向靶场交互式评测、代理式任务评测和多阶段攻防流程评测扩展，形成从知识理解到行动执行的多层次评测体系。

其次，本文已经在能力维度上对模型差异进行了较细致分析，但现有维度划分仍主要围绕 ATT&CK 知识结构和题目设计需求展开，在更广泛的安全任务场景中仍有继续细化的空间。例如，后续可以进一步区分模型在漏洞成因分析、攻击链补全、防御规则生成、告警解释和处置建议等更具体子能力上的表现，并探索这些能力之间的相关性与迁移关系，从而使评测结果更贴近实际安全工作流。

再次，针对开放式题型的自动评分方法，本文采用了以技术编号提取和归一化匹配为核心的评分思路。这种方法能够较好解决当前正式实验中的自动化评分需求，但对于回答中存在的隐式表达、语义近似、步骤顺序合理但表述不标准等情况，仍可能存在区分能力不足的问题。未来可以考虑引入更稳定的层次化评分机制，例如先进行结构化技术点抽取，再结合语义一致性约束或人工抽样复核，以进一步提升复杂推理题型评分的精细度与可信度。

最后，本文目前的实验对象仍以少量代表性模型为主，虽然已经能够初步反映主流模型在渗透测试知识任务中的能力差异，但在模型规模、训练路线、推理模式和服务接口方面仍缺少更系统的横向覆盖。后续研究可以在统一实验框架下纳入更多开源模型、推理增强模型以及面向安全场景优化的专用模型，并持续扩展正式冻结数据集，从而形成更具代表性和持续更新能力的评测基准。

综上所述，本文的研究工作为大语言模型渗透测试知识能力评估提供了一个可落地的起点，但围绕真实任务建模、能力维度细化、开放题评分优化和基准持续扩展等方向，仍然存在较大的深化空间。随着大语言模型在网络安全领域中的应用不断加深，构建更贴近真实场景、更强调能力结构解释力的评测体系，将具有持续而重要的研究价值。
